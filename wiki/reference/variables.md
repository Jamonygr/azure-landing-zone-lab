# Variables reference

The generated [root Terraform reference](../../TERRAFORM.md#inputs) is the
canonical list of input types, defaults, sensitivity, and requirements.

Important interface rules:

- `environment` is one of `cheap-lab`, `lab`, `dev`, or `prod`.
- `repository_url` is optional; the Repository tag is omitted when it is null.
- `deploy_private_endpoints` requires `deploy_private_dns_zones = true`.
- `appgw_waf_mode` is `Detection` or `Prevention`.
- Start/stop schedules use 24-hour `HH:MM` values.
- Regulatory remediation roles default to none and must be supplied as full,
  least-privilege role-definition IDs.
- VM, SQL, and VPN secrets should be supplied through ephemeral `TF_VAR_*`
  environment variables or allowed to be generated by Terraform.

Use the tracked profiles in `environments/` for non-secret configuration.
`prod.tfvars` is a production-like learning profile, not a production baseline.
